# Deutsches Schreiben: Anti-KI-Referenz

Diese Referenz listet alle bekannten Erkennungszeichen KI-generierter deutscher Texte und die verbindlichen Gegenregeln: Wortlisten, Floskeln, Satzbau, Struktur, Interpunktion, E-Mail/Slack und Übersetzungsartefakte. Sie gilt für jeden deutschen Text-Output, vom Blogartikel über das Angebot bis zur Zwei-Zeilen-Mail.

Grundprinzip: KI rät statistisch das Nächstwahrscheinliche, das auf möglichst viele Fälle passt. Das Gegenmittel ist immer Spezifik: Zahlen, Namen, Daten, echte Fälle, eine Position. Struktur-Regelmäßigkeit verrät mehr als einzelne Wörter, deshalb zuerst den Rhythmus fixen, dann das Vokabular.

---

## 1. Wortlisten

Diese Wörter kommen in deutschem KI-Output um ein Vielfaches häufiger vor als in menschlichen Texten. Sie treten in Clustern auf: Wo eines steht, stehen meist mehrere. Einzelne Treffer sind erlaubt, Häufung nicht. Faustregel: die Hälfte aller Adjektive streichen, jedes gestrichene Buzzword durch das Konkretum ersetzen, das es verdeckt.

### 1.1 Adjektive

| Tell | Stattdessen |
|---|---|
| entscheidend, essenziell, wesentlich, maßgeblich, zentral | streichen oder Ursache-Wirkung nennen („ab 3 Sekunden Ladezeit springen 40 % ab") |
| ganzheitlich, umfassend | aufzählen, was konkret enthalten ist |
| maßgeschneidert, individuell, passgenau | das konkret Anpassbare nennen („drei Paketgrößen, monatlich kündbar") |
| nahtlos | den konkreten Übergang beschreiben oder streichen |
| innovativ, revolutionär, bahnbrechend, wegweisend, zukunftsweisend | sagen, was neu ist (Funktion, Jahr, Unterschied) |
| leistungsstark, robust, effizient, effektiv | Messwert („rendert 4K-Video in 8 Minuten") |
| hochwertig, erstklassig, einzigartig, unverzichtbar | prüfbares Qualitätsmerkmal oder Beleg |
| spannend, faszinierend, beeindruckend, atemberaubend | streichen; der Inhalt muss es selbst zeigen |
| vielfältig, facettenreich | die Varianten konkret aufzählen |
| dynamisch, nachhaltig (als Füllwort) | streichen oder präzisieren (ökologisch? dauerhaft? Zahl?) |
| optimal, vielversprechend, nuanciert | Position mit Begründung oder streichen |

### 1.2 Verben

| Tell | Stattdessen |
|---|---|
| eintauchen („Tauchen wir ein in die Welt des ...") | ersatzlos streichen, direkt mit dem Inhalt anfangen |
| beleuchten, ergründen | zeigen, erklären, oder gleich den Inhalt liefern |
| revolutionieren, transformieren | konkret: was ändert sich, für wen, ab wann |
| optimieren (inflationär) | das genaue Verb: beschleunigen, kürzen, verdoppeln |
| ermöglichen, gewährleisten, sicherstellen (inflationär) | aktives Verb mit handelndem Subjekt |
| fördern, stärken (als Kalque von foster) | konkret sagen, was aufgebaut wird |
| navigieren, adressieren, meistern (Probleme) | lösen, angehen |
| entfesseln, Potenzial entfalten, aufs nächste Level heben | das messbare Ergebnis nennen |
| punkten mit, bestechen durch, überzeugen durch, glänzen mit, aufwarten mit | ist/hat + Fakt |
| bieten („Wir bieten ...") | machen: „Wir bringen Ihre Seite in die Top 10 für X" |
| fungieren als, dienen als, stellt dar, zeichnet sich aus durch | ist, hat (Kopula nicht vermeiden, siehe 3.7) |

### 1.3 Substantive und Nebelwörter

| Tell | Stattdessen |
|---|---|
| digitale Landschaft, Marketinglandschaft | das konkrete Ding: „Google, Instagram und der Onlineshop" |
| Reise („Ihre Reise beginnt hier"), die Welt des X | streichen |
| Herausforderungen | das konkrete Problem („zu wenig Anfragen über die Website") |
| Mehrwert | was der Leser danach kann oder hat |
| Synergien, Gamechanger, Booster | den konkreten Effekt beschreiben |
| Meilenstein (inflationär) | das Ereignis nüchtern nennen |
| ganzheitlicher Ansatz | die Bestandteile aufzählen |
| Aspekt, Faktor, Bereich, Ebene, Thema („rund um das Thema X") | das Ding direkt benennen |
| Lösung(en) (inflationär) | Produkt oder Leistung beim Namen nennen |
| Potenzial, Möglichkeiten | konkrete Optionen aufzählen |
| Fundament, Kompass, Werkzeugkasten, Stellschrauben, Hebel, Dreh- und Angelpunkt | wörtlich schreiben (siehe 2.8 Metaphern) |
| im digitalen Zeitalter, in der heutigen Zeit, Ära | streichen |

### 1.4 Konnektoren

Gehobene Konnektoren sind der zuverlässigste deutsche Wort-Tell (Übersetzungen von moreover/furthermore/additionally). Maximal ein gehobener Konnektor pro Seite; guter Text braucht kaum explizite Verknüpfung, die Logik trägt.

| Tell | Stattdessen |
|---|---|
| zudem, darüber hinaus, des Weiteren, ferner, überdies, obendrein | „und", „auch", „dazu kommt" oder Sätze direkt aneinanderreihen |
| nicht zuletzt, hierbei, dabei (inflationär) | streichen |
| somit, folglich, demnach, dementsprechend | „also", „deshalb", oder die Logik unausgesprochen tragen lassen |
| letztendlich, schlussendlich | „am Ende", „letztlich" (sparsam) |
| sowie (als Dauer-und) | „und" |
| jedoch (als Standard-Kontrast in jedem zweiten Absatz) | „aber" |
| nichtsdestotrotz | „trotzdem" |
| in Bezug auf, hinsichtlich, bezüglich, diesbezüglich, seitens, im Rahmen, im Zuge, im Hinblick auf | Amtsdeutsch auflösen: konkret sagen, worum es geht |

### 1.5 Denglisch-Buzzwords

Maximal 1 bis 2 etablierte Anglizismen pro Text. Echte Fachsprache (SEO, Content, Ranking, Conversion, Snippet) bleibt englisch; nie zwanghaft eindeutschen (kein „Verlobungsköder" für engagement bait).

| Tell | Stattdessen |
|---|---|
| Learnings | Erkenntnisse, Lehren |
| Insights | Befunde, Erkenntnisse |
| Painpoints | Probleme |
| Touchpoints | Kontaktpunkte |
| Deep Dive | Analyse |
| Journey | Kaufprozess, Ablauf (konkret) |
| Mindset, Approach, Alignment, Commitment, Outcome, Impact | deutsche Entsprechung oder streichen |
| Best Practices (inflationär) | „was funktioniert", bewährte Vorgehen |
| State of the Art, seamless, on point, Content-Pieces | streichen oder konkret formulieren |
| „Lass uns connecten", „excited zu sharen", „Content droppen" | idiomatisch deutsch, wie im Gespräch mit einem Kunden |

## 2. Floskeln

### 2.1 Weltlage-Opener

**Verbot:** Kein Text beginnt mit einer Epochendiagnose. Die sichersten deutschen KI-Marker überhaupt: „In der heutigen digitalen Welt", „In der heutigen schnelllebigen (Geschäfts-)Welt", „In einer Welt, in der ...", „In Zeiten von ...", „Im digitalen Zeitalter", „Immer mehr Menschen/Unternehmen ...", „... ist wichtiger denn je", „Tauchen wir ein", „Werfen wir einen Blick auf", „Herzlich willkommen" (als Blog-Opener), „Kennen Sie das?", „Stellen Sie sich vor, ..." (als Standard-Opener).

**Stattdessen:** Ersten Absatz komplett streichen und mit dem konkretesten Satz des Textes starten: Zahl, Fall, Behauptung oder Problem. Beispiel: „Die Zahnklinik bekam 2025 mehr Patientenanfragen über Google als über alle anderen Kanäle zusammen." Test: Passt der erste Satz auch über jedes andere Thema? Dann weg. Die Einleitung ist statistisch der KI-verdächtigste Textteil; immer selbst formulieren oder radikal kürzen.

### 2.2 Meta-Ankündigungen

**Verbot:** Sätze, die beschreiben, was der Text gleich tut: „In diesem Artikel erfahren Sie ...", „Im Folgenden zeigen wir Ihnen ...", „Dieser Beitrag beleuchtet ...", „Doch dazu später mehr", „Lassen Sie uns die wichtigsten Punkte betrachten". Auch erste Sätze, die nur die Überschrift paraphrasieren (Überschrift „Vorteile von X", erster Satz „X hat viele Vorteile.").

**Stattdessen:** Machen statt ankündigen. Der Inhalt beginnt im ersten Satz. Wenn Orientierung nötig ist: knappes Inhaltsverzeichnis, kein Prosa-Ankündigungssatz.

### 2.3 Es-ist-wichtig-Gerüst

**Verbot:** Unpersönliche Wichtigkeits-Behauptungen: „Es ist wichtig zu beachten, dass ...", „Es ist erwähnenswert/bemerkenswert, dass", „Es gilt zu beachten", „Es versteht sich von selbst", „Es lässt sich nicht leugnen", „Es empfiehlt sich", „Beachten Sie, dass", „Denken Sie daran:", inflationäre „Wichtig:"-Einschübe.

**Stattdessen:** Wichtigkeit durch Inhalt zeigen: „Die Ladezeit ist ein Rankingfaktor. Ab 3 Sekunden springen 40 % ab." Jeder Wichtig-Satz bekommt eine Ursache-Wirkung-Begründung oder fliegt raus.

### 2.4 Fazit-Formeln

**Verbot:** „Zusammenfassend lässt sich sagen", „Abschließend lässt sich festhalten", „Insgesamt zeigt sich", „Im Großen und Ganzen", „Alles in allem", „Unterm Strich" (inflationär). Ebenso der Zwang, jeden Text mit einem Abschnitt „Fazit" zu beenden, der nur Gesagtes wiederholt, und der Zukunftsausblick-Schluss („Es bleibt spannend zu beobachten, wie ...", „Die Zukunft wird zeigen ...", „Wir blicken optimistisch in die Zukunft").

**Stattdessen:** Mit dem stärksten Punkt, einer Entscheidung oder einer konkreten Handlung enden, nie mit einer Wiederholung des Gelesenen. Wenn ein Schluss fachlich nötig ist: neue Synthese oder Empfehlung mit Kante („Unter 5.000 € Budget zuerst die drei lokalen Landingpages, erst danach den Blog"). Die H2 nicht „Fazit" nennen, sondern die Empfehlung selbst („Womit Sie diese Woche anfangen").

### 2.5 Bedeutungs-Behauptungen

**Verbot:** „spielt eine wichtige/entscheidende/zentrale Rolle", „ist von entscheidender Bedeutung", „gewinnt zunehmend an Bedeutung", „unterstreicht die Bedeutung von", „ist ein Zeugnis für", „ist nicht zu unterschätzen", „markiert einen Wendepunkt". KI pumpt die Wichtigkeit jedes Themas auf, selbst bei Banalitäten.

**Stattdessen:** Sagen, was die Sache konkret bewirkt: „Ohne passende Keywords findet Google die Seite für keine relevante Suche." Streich-Test: Funktioniert der Satz ohne die Bedeutungs-Klausel noch? Dann Klausel löschen; die Bewertung macht der Leser.

### 2.6 Agentur- und Marketing-Bausteine

**Verbot:** Die Standardbausteine, die jede KI-Landingpage identisch machen: „maßgeschneiderte Lösungen", „ganzheitlicher Ansatz", „Ihr volles Potenzial entfalten", „auf das nächste Level bringen", „Profitieren Sie von", „Ihre Vorteile auf einen Blick", „alles aus einer Hand", „Ihr (zuverlässiger) Partner für", „alles rund um das Thema X", „Egal ob ... oder ... – wir sind für Sie da", „Wir freuen uns auf Sie!", „Überzeugen Sie sich selbst". Durchschnitts-Headlines („Entfesseln Sie Ihr Potenzial", „Ihre All-in-one-Lösung", „Digitalisierung. Neu gedacht."), generische CTAs („Jetzt loslegen", „Mehr erfahren", „Jetzt unverbindlich anfragen" als einziger CTA) und Meta-Description-Slop („Erfahren Sie jetzt alles Wichtige ...", „Jetzt entdecken!"). In Angeboten: der Einstieg „Vielen Dank für Ihr Interesse. Gerne unterbreiten wir Ihnen folgendes Angebot" plus Selbstlob-Prosa („führende Agentur", „langjährige Erfahrung") und vage Sammelposten („Ganzheitliche SEO-Betreuung", „Laufende Optimierung").

**Stattdessen:** Spezifik schlägt Anspruch: eine Persona, eine Leistung, ein prüfbares Ergebnis. CTA benennt, was passiert: „Schicken Sie uns Ihre URL, Sie bekommen in 48 h drei Quick Wins." Angebote steigen über die Situation des Kunden ein (konkreter Befund aus dem Erstgespräch); jede Leistungsposition mit Menge, Frequenz und Deliverable („4 Kategorietexte/Monat, je 800 bis 1.200 Wörter, inkl. 1 Korrekturschleife") plus expliziter Abgrenzung, was nicht enthalten ist. Meta-Descriptions mit mindestens einer Zahl oder prüfbaren Aussage, unique pro Seite.

### 2.7 Titel-Schablonen

**Verbot:** „Der ultimative Guide für X", „Alles, was Sie über X wissen müssen", „X: Warum Y jetzt wichtiger ist denn je", „So gelingt X: Die besten Tipps", „10 Tipps, die wirklich funktionieren", „X leicht gemacht", „Ein umfassender Leitfaden", „Das müssen Sie wissen". Doppelpunkt-Zweiteiler als Dauerform; alle H2 gleich lang und gleich gebaut; dramatische Nominalphrasen-Headings im Filmposter-Stil statt Inhaltsangabe.

**Stattdessen:** Titel mit konkreter Behauptung, Zahl aus dem Inhalt oder Standpunkt: „Zahnimplantat-Kosten 2026: reale Preise aus 40 Behandlungsplänen (1.800 bis 3.400 €)". Test: Könnte der Titel wortgleich über jedem Konkurrenz-Artikel stehen? Dann verwerfen. Muster variieren, nicht jede Überschrift als Doppelpunkt-Zweiteiler.

### 2.8 Metaphern-Katalog

**Verbot:** Das KI-Metaphern-Repertoire: „wie ein Leuchtturm im Nebel", „der Kompass für ...", „das Fundament, auf dem ...", „kein Sprint, sondern ein Marathon", „die Stellschrauben", „der Werkzeugkasten", „das Puzzleteil", „unter der Haube", „Brücken bauen", „Türöffner". Besonders verräterisch: mehrere gemischte Metaphern pro Absatz.

**Stattdessen:** Maximal eine Metapher pro Text, und nur wenn sie etwas erklärt, das wörtlich schwerer wäre. Frische, fachnahe Bilder oder schlicht wörtlich schreiben.

## 3. Satzbau und Rhythmus

### 3.1 Dreierregel brechen

**Verbot:** Durchgängige Dreiergruppen: drei Adjektive („intuitiv, schnell und effizient"), drei Beispiele, drei Vorteile, drei Bullets, „sowohl ... als auch ... und ...". In Kombination mit Gedankenstrich („– sicher, garantiert und zweifellos") der auffälligste Stil-Tell. Auch symmetrische Gliederung: 3 Hauptpunkte mit je 3 Unterpunkten.

**Stattdessen:** Ein präzises Adjektiv statt drei vager. Aufzählungen mit 2, 4 oder 5 ungleich gewichteten Punkten. Wenn wirklich drei Dinge relevant sind: eins ausführen, zwei kurz erwähnen. Maximal eine Adjektiv-Triade pro Text.

### 3.2 Negative Parallelismen

**Verbot:** „Nicht nur ..., sondern auch ..." mehr als einmal pro Text. Ebenso das leere Umdeutungs-Muster: „Es ist nicht X. Es ist Y.", „Es geht nicht um X, sondern um Y", „X ist weit mehr als nur Y", „Das ist kein Trend. Das ist eine Revolution." Klingt profund, sagt nichts, und tut so, als räume es ein Missverständnis aus, das niemand hatte. Auch abgehängte Verneinungs-Fragmente („..., kein Rätselraten.").

**Stattdessen:** Y direkt behaupten: „SEO ist vor allem Strategie." Wenn ein Kontrast bleibt: mit Beleg differenzieren, was sich konkret ändert, für wen, ab wann. Verneinungs-Fragmente als vollständigen Satz ausformulieren.

### 3.3 Frage-Antwort-Fragmente und Staccato-Parataxe

**Verbot:** Dramatische Fragmente im Punchline-Stil: „Das Ergebnis? Mehr Umsatz.", „Die gute Nachricht? Es geht einfacher.", „Klingt vertraut?". Ebenso Staccato-Ketten: drei oder mehr aufeinanderfolgende Kurzsätze unter 10 Wörtern in Parallelstruktur („Fünf Tools. Fünf Tabs. Fünf Probleme."), rhetorische Fragen als Sektions-Übergang („Was heißt das für Sie?") und Fake-Vertraulichkeit als Opener („Ganz ehrlich?", „Die Wahrheit ist:", „Hier ist die Sache:"). Nicht jeder Absatz darf mit einer Punchline enden; klingt ein Satz wie eine Zitat-Kachel, ist er verdächtig.

**Stattdessen:** Als normalen Aussagesatz schreiben („Das Ergebnis ist Burnout."). Verwandte Gedanken mit Nebensätzen und Konjunktionen verbinden, sodass die Syntax zeigt, wie Ideen zusammenhängen. Das Stilmittel Kurzfragment maximal einmal pro Text, an der stärksten Stelle. Wer die Antwort auf eine rhetorische Frage kennt, sagt sie einfach.

### 3.4 Satzlängen-Varianz erzwingen

**Verbot:** Der KI-Korridor: fast jeder Satz 15 bis 25 Wörter mit identischem Bauplan (Hauptsatz, Komma, Nebensatz). Drei gleich lange Sätze in Folge. Monotone Satzanfänge (jeder Satz beginnt mit Subjekt-Verb oder „Sie können"). Gleichförmigkeit ist das stärkste Detektionssignal überhaupt, stärker als jedes Einzelwort.

**Stattdessen:** Bewusst mischen: etwa 30 % kurze Sätze (bis 8 Wörter), 60 % mittlere, 10 % lange. Ein-Wort- und Fragment-Sätze sind erlaubt. Genau. So. Satzanfänge variieren: Adverb, Objekt, Nebensatz, Frage oder Zitat voranstellen. Nach jedem Absatz prüfen: drei gleich lange Sätze in Folge? Einen brechen.

### 3.5 Nominalstil, Passiv, Amtsdeutsch

**Verbot:** Behördenstil-Ketten: „Die Implementierung von Strategien zur Optimierung der Effizienz", „Die Durchführung der Maßnahme erfolgt ...", „Es wird empfohlen ...". Erkennbar an -ung/-ierung/-keit-Ketten, Ersatzverben (erfolgen, stattfinden, vorgenommen werden), Genitivketten und verursacherlosem Passiv.

**Stattdessen:** Aktiv, Subjekt-Verb-Objekt: „Wir schreiben den Text für Zielgruppe X." Jede -ung-Kette in ein Verb zurückverwandeln. Wer handelt, steht im Satz; wenn keine Person passt, den Leser ansprechen. „Im Rahmen der Optimierung" wird „beim Optimieren" oder entfällt.

### 3.6 Partizipial-Anglizismen

**Verbot:** Aus dem Englischen kopierte -end-Konstruktionen und nachgestellte Pseudo-Analysen: „Das Tool automatisiert Berichte, wertvolle Zeit sparend.", „..., was die Bedeutung von X unterstreicht" als beleg-loser Nachklapp. Im Deutschen unidiomatisch bis falsch.

**Stattdessen:** Zwei Hauptsätze oder Relativsatz: „Das Tool automatisiert Berichte. Das spart pro Woche etwa zwei Stunden." Kausal-Behauptungen („und stärkte damit ...") nur mit Beleg, sonst streichen.

### 3.7 Kopula nicht vermeiden

**Verbot:** Pressemitteilungs-Verben statt „ist/hat": „fungiert als", „dient als", „stellt dar", „bietet", „verfügt über", „zeichnet sich aus durch", „präsentiert sich als". KI weicht systematisch auf gespreizte Verben aus.

**Stattdessen:** „ist" und „hat" als Default. Ein spezifischeres Verb nur, wenn es echte Bedeutung hinzufügt. „Die Galerie verfügt über vier Räume" wird „Die Galerie hat vier Räume."

### 3.8 Hedging und Pauschalisierung

**Verbot:** Absicherungs-Kaskaden ohne Festlegung: „könnte", „in vielen Fällen", „je nach Anwendungsfall", „grundsätzlich", „in der Regel", „unter Umständen", „tendenziell", „hängt von verschiedenen Faktoren ab und sollte individuell geprüft werden". Ebenso das Gegenteil, faule Extreme als vage Autorität: „jeder", „immer", „nie", „alle wissen". Und das Hedging-Seesaw: Pro und Contra gleichgewichtig abwägen, ohne Position.

**Stattdessen:** Festlegen: „Das funktioniert." / „Für Shops unter 500 Produkten lohnt sich das nicht." Bedingungen konkret machen (wann gilt es, wann nicht) statt pauschal relativieren. Maximal ein Hedge pro Aussage; Gegenposition in einem Nebensatz, nicht als Gleichgewicht. Pro Abschnitt eine Entscheidung: „Tu X, nicht Y."

### 3.9 Synonym-Karussell stoppen

**Verbot:** Zwanghafte Variation für dasselbe Ding: „das Tool, die Lösung, die Software, die Anwendung, die Plattform" in einem Absatz. Umgekehrt wiederholt KI Argumente paraphrasiert (dieselbe Aussage zwei- bis dreimal in anderen Worten).

**Stattdessen:** Ein Begriff pro Konzept, konsequent durchgehalten; Menschen wiederholen das klarste Wort. Inhaltliche Wiederholungen komprimieren: drei paraphrasierte Sätze zu einem verdichten.

### 3.10 Der aber-Test

**Verbot/Tell:** Ein deutscher Text ohne ein einziges „aber" ist fast immer maschinell. KI weicht auf „jedoch", „allerdings", „nichtsdestotrotz" aus. Englischsprachige Prüfregeln erwischen das nie.

**Stattdessen:** „aber" aktiv verwenden, die jedoch-Quote deckeln. Auch Sätze mit „Und", „Aber", „Denn" beginnen, wo es dem Sprachfluss dient.

### 3.11 Fake-Bandbreiten

**Verbot:** Schein-Breite ohne Skala: „Von der Strategie über die Umsetzung bis hin zum Reporting", „Von Startups bis zu Konzernen", „Egal ob ... oder ...", „Ganz gleich, ob ...". Klingt umfassend, benennt nichts und meint „alle".

**Stattdessen:** Die tatsächlichen Leistungen oder die eine echte Zielgruppe nennen. Wer für alle schreibt, schreibt für niemanden.

## 4. Struktur und Formatierung

### 4.1 Zwischenüberschriften-Inflation

**Verbot:** Jeder Gedanke bekommt eine H2/H3; mehr als 3 Überschriften auf 300 Wörter; alle Überschriften nach identischem Muster („Warum X wichtig ist", „Wie Sie X optimieren", alle als Frage oder alle als Doppelpunkt-Schema). Erster Satz unter der Überschrift wiederholt nur die Überschrift. Keyword mechanisch in jede H2 gepresst („Zahnreinigung Kosten / Zahnreinigung Ablauf / Zahnreinigung Vorteile").

**Stattdessen:** Überschriften nur bei echtem Themenwechsel, unterschiedlich lang und gebaut, mit Information statt Muster („Ladezeit: ab 3 Sekunden verlieren Sie 40 %"). Der erste Satz danach bringt neue Information. Hauptkeyword in maximal 1 bis 2 von n Zwischenüberschriften.

### 4.2 Schema-F-Gliederung

**Verbot:** Die Ratgeber-Schablone „Was ist X? / Warum ist X wichtig? / Vorteile von X / Nachteile / Herausforderungen / Fazit / FAQ" mit definitorischem Einstieg, symmetrischem Vorteil-Nachteil-Paar und Pflicht-Herausforderungen-Abschnitt („Trotz aller Vorteile gibt es auch Herausforderungen"). Alle Abschnitte gleich lang.

**Stattdessen:** Gliederung aus der Sache und der Entscheidungslogik des Lesers ableiten: mit dem Problem oder Fall starten, Reihenfolge nach Relevanz. Mindestens eine H2 bezieht Position („Wovon wir abraten", „Der häufigste Fehler"). Asymmetrie zulassen: ein Abschnitt darf fünfmal so lang sein wie ein anderer.

### 4.3 Mini-Fazits und Absatz-Inseln

**Verbot:** Jeder Abschnitt endet mit einer Zusammenfassung („Somit ist die Keyword-Recherche ein wichtiger erster Schritt.") und der nächste beginnt ohne Bezug. Alle Absätze gleich lang, identischer Innenaufbau (These, zwei Belege, Mini-Fazit), alles auf derselben Abstraktionsebene ohne ein einziges Detail.

**Stattdessen:** Abschnitte dürfen offen enden und aufeinander verweisen („Das rächt sich beim nächsten Punkt:"). Absatzlängen variieren, Ein-Satz-Absätze erlaubt. Pro Abschnitt mindestens ein konkretes Detail, Beispiel oder eine Zahl; die Abstraktionsebene wechseln. Manche Absätze dürfen abrupt aufhören.

### 4.4 FAQ-Inflation

**Verbot:** Der Pflicht-FAQ-Block am Ende jedes Artikels mit generischen Fragen, die im Text schon beantwortet wurden, oder Fragen in Keyword-Form, die niemand so stellt.

**Stattdessen:** FAQ nur mit nachweislich echten Fragen (aus Kundenmails, Support, Suchanfragen), deren Antworten neue Information enthalten. Sonst streichen.

### 4.5 Listen-Fetisch

**Verbot:** Texte, die überwiegend aus Aufzählungen bestehen; exakt 3 oder 5 Punkte, alle gleich lang und gleich gebaut; das Stanzmuster „**Effizienz:** Beschreibungssatz" (gefetteter Begriff, Doppelpunkt, ein Satz) in Serie; Emoji-Bullets; nummerierte Schritt-Ketten, wo keine Reihenfolge existiert; Listen, die auf eine runde Zahl aufgefüllt wurden („Die 7 wichtigsten ...").

**Stattdessen:** Listen nur für echt listenartige Inhalte (Checklisten, Schritte mit Reihenfolge, Optionen). Argumente als Fließtext mit Gewichtung. Wenn Liste: ungleich lange Punkte, keine Emoji-Bullets, eine Meinung dazu (häufigster Fehler, was zuerst). Mehr als 7 Bullets in Folge auflösen.

### 4.6 Fettdruck-Inflation

**Verbot:** Mehrere gefettete Halbsätze pro Absatz als Pseudo-Betonung („Dabei ist es **besonders wichtig**, die **richtige Strategie** zu wählen"). In Kombination mit Emoji-Überschriften und Markdown-Resten im CMS ein sicherer Tell.

**Stattdessen:** Maximal eine Fettung pro Bildschirmseite, nur für die eine Kernaussage oder eine Zahl. Wenn etwas wichtig genug fürs Fetten ist: Satz so umbauen, dass es vorne steht.

### 4.7 Dichte statt Wortzahl

**Verbot:** Auf Ziel-Wortzahl strecken: derselbe Fakt in Einleitung, Hauptteil und Schluss umformuliert; Füllabsätze ohne Information („Es ist wichtig zu beachten, dass die Wahl des richtigen Anbieters eine entscheidende Rolle spielt."); aufgeblasene Überleitungen.

**Stattdessen:** Jeder Absatz trägt eine neue, konkrete Information (Zahl, Beispiel, Anweisung, Beleg). Streich-Test im Review: Was ohne Informationsverlust löschbar ist, fliegt raus. 600 dichte Wörter schlagen 1.800 gestreckte.

### 4.8 Scaffold-Überschriften

**Verbot:** Generische Gerüst-Header als Default: „Einleitung", „Überblick", „Fazit", „Ausblick", „Key Takeaways", „Das Wichtigste in Kürze" nach jedem Abschnitt. Executive Summaries ohne eine einzige Zahl. Restate-Satz direkt nach der Überschrift.

**Stattdessen:** Sprechende Überschriften, die den Befund vorwegnehmen. Höchstens eine Zusammenfassung pro Dokument, am Anfang, mit den drei wichtigsten konkreten Befunden samt Zahlen und einer Empfehlung. Testfrage: Könnte diese Summary über jedem beliebigen Report stehen? Dann neu schreiben.

## 5. Interpunktion und deutsche Typografie

### 5.1 Gedankenstrich

**Verbot:** Der englische Geviertstrich „—" ohne Leerzeichen ist im Deutschen doppelt falsch: falsches Zeichen und KI-Marker Nummer eins. Auch der korrekte Gedankenstrich wird von KI weit häufiger gesetzt als von deutschen Autoren, oft mehrfach pro Absatz, gern vor Adjektiv-Triaden („– sicher, garantiert und zweifellos").

**Stattdessen:** Höchstens 1 bis 2 Gedankenstriche pro Text, dann korrekt deutsch: Halbgeviertstrich mit Leerzeichen („Wort – Wort"). Ersatzreihenfolge: Punkt (neuer Satz), Komma, Doppelpunkt, Klammern, Satz umbauen. Suchlauf nach „—" vor jeder Abgabe.

### 5.2 Anführungszeichen und Apostroph

**Verbot:** Englische Anführungszeichen ("..." oder “...”) statt deutscher „..." bzw. »...«; gemischte Stile im selben Text; Deppenapostroph („Anna's Blog"); gerades Ersatzzeichen (') statt typografischem Apostroph (’).

**Stattdessen:** Vor Abgabe alle Anführungszeichen auf den deutschen Hausstil normalisieren; in Code und Slack gerade Zeichen konsistent verwenden. Genitiv ohne Apostroph („Annas Blog"). In Templates und PDF-Pipelines die deutschen Zeichen fest hinterlegen.

### 5.3 Zahlen, Prozent, Datum, Währung

**Verbot:** Übersetzungs-Artefakte: Dezimalpunkt („3.5 Sekunden"), Tausender-Komma („10,000"), englische Datumsformate („Januar 5, 2026", „01/05/2026"), vorangestellte Währung („EUR 500"), inkonsistente Prozent-Schreibweise im selben Text.

**Stattdessen:** Konsistenz-Pass vor Abgabe: Dezimalkomma (3,5), Tausenderpunkt (10.000), deutsches Datum (5. Januar 2026), nachgestelltes Euro-Zeichen (500 €), geschütztes Leerzeichen vor % und Einheiten, ein Format pro Text.

### 5.4 Title Case, serielles Komma, Durchkopplung

**Verbot:** Englische Konventionen in deutschen Texten: Title Case in Überschriften („Die Besten Tipps Für Ihr Online-Marketing"), Oxford-Komma („schnell, einfach, und günstig"), Deppenleerzeichen in Komposita („Content Marketing Strategie").

**Stattdessen:** Deutsche Groß- und Kleinschreibung in Überschriften (nur Satzanfang und Substantive), kein Komma vor „und", Komposita durchkoppeln („Content-Marketing-Strategie", „Social-Media-Marketing-Agentur").

### 5.5 Interpunktions-Palette und Ausrufezeichen

**Verbot/Tell:** Die KI-Signatur: viele Gedankenstriche, aber kaum Semikolons, kaum Klammern, hundertprozentig konsistente Zeichensetzung, keine Kontraktionen, keine Normabweichung. Ausrufezeichen als Enthusiasmus-Ersatz.

**Stattdessen:** Die volle Palette mischen: gelegentlich ein Semikolon, ein Klammer-Einschub, eine Doppelpunkt-Pointe. Maximal ein Ausrufezeichen pro 1.000 Wörter; Begeisterung kommt aus der Wortwahl. Gesprochene Formen zulassen, wo das Register es erlaubt („gibt's"). Keine künstlichen Tippfehler einbauen; Echtheit kommt aus dem Inhalt, nicht aus simulierter Schlampigkeit.

### 5.6 Markdown- und Unicode-Artefakte

**Verbot:** Markdown im falschen Medium: sichtbare **Sternchen**, ##-Überschriften, [Text](URL)-Links oder ---Trenner in CMS, Word, Mails oder Slack; übersprungene Überschriften-Ebenen; unsichtbare Unicode-Zeichen (Zero-Width-Spaces) aus dem LLM-Output; Leerzeichen am Absatzanfang.

**Stattdessen:** Vor dem Einfügen ins Zielsystem konvertieren und im Zielformat rendern und prüfen. In Plaintext-Kanälen komplett auf Markdown verzichten.

## 6. Übersetzungs-Artefakte

### 6.1 Kalques (wörtlich übersetzte Idiome)

**Verbot:** Sätze, die auf Englisch natürlicher klingen als auf Deutsch. Rückübersetzungs-Test: Jeden verdächtigen Satz mental ins Englische übersetzen; klingt er dort besser, ist er ein Kalque und wird idiomatisch deutsch neu formuliert.

| Tell | Stattdessen |
|---|---|
| Sinn machen | Sinn ergeben |
| in 2026 | 2026 / im Jahr 2026 |
| Am Ende des Tages | letztlich, unterm Strich (sparsam) |
| die Extrameile gehen | mehr tun als nötig (oder konkret) |
| das Beste aus beiden Welten | die Vorteile konkret nennen |
| der Elefant im Raum | das offensichtliche Problem benennen |
| Seien wir ehrlich / Lassen Sie uns ehrlich sein | streichen, einfach das Ding sagen |
| Hier ist die Sache / Die Wahrheit ist: | streichen |
| Sagen Sie Hallo zu X / Verabschieden Sie sich von X | direkt beschreiben, was sich ändert |
| Sie sind nicht allein / Keine Sorge | streichen |
| Schnallen Sie sich an / Spoiler: / Fun Fact: / Pro-Tipp: | streichen |
| Das Beste daran? / Raten Sie mal | Aussagesatz |
| Lass das mal sacken / Lies das nochmal | ersatzlos streichen; starke Sätze wirken ohne Regieanweisung |
| Unpopuläre Meinung: | Position einfach vertreten |
| realisieren (im Sinn von erkennen) | erkennen, merken |
| mehr und mehr | immer mehr |
| einmal mehr | wieder, erneut |
| nicht wirklich | eigentlich nicht, kaum |

### 6.2 Satzstellung und Adverb-Position

**Verbot:** Englische Wortstellung in deutschen Sätzen: Verb nicht am Ende des Nebensatzes („..., weil Google bewertet die Ladezeit"), Adverbien an englischer Position („Wir haben erfolgreich das Projekt abgeschlossen"), lange lineare Sätze ohne deutsche Klammerstruktur.

**Stattdessen:** Laut lesen: Stolperstellen sind meist englische Syntax. Nebensatz-Verben ans Ende, Satzklammer nutzen, Adverbien an die natürliche deutsche Position („Wir haben das Projekt erfolgreich abgeschlossen").

## 7. E-Mail und Slack

### 7.1 Verbotene Opener

| Tell | Stattdessen |
|---|---|
| Ich hoffe, diese E-Mail erreicht Sie gut | Direkt mit dem Anlass starten: „Kurze Frage zu X:" |
| Ich hoffe, es geht Ihnen gut (ohne Bezug) | weglassen oder echter Bezug („Danke für das Gespräch am Dienstag") |
| Ich hoffe, Sie hatten ein schönes Wochenende (ohne Bezug) | weglassen |
| Gerne möchte ich mich bei Ihnen melden bezüglich ... | „Kurzes Update zum Relaunch: Wir sind 3 Tage vor Plan." |
| Bezugnehmend auf Ihr Schreiben | „Zu Ihrer Mail vom 3. Juli:" |
| Vielen Dank für Ihre wunderbare Frage! | Substanz statt Lob, direkt antworten |

### 7.2 Verbotene Closer

| Tell | Stattdessen |
|---|---|
| Zögern Sie nicht, mich zu kontaktieren | konkreter nächster Schritt |
| Für Rückfragen stehe ich jederzeit gerne zur Verfügung (Pflichtschluss) | „Passt Donnerstag 14 Uhr für 15 Minuten?" |
| Vielen Dank im Voraus für Ihre Bemühungen | konkrete Bitte mit Datum |
| Ich hoffe, das hilft | weglassen |
| Lassen Sie es mich wissen, falls Sie Fragen haben | genau eine konkrete Frage stellen |
| Gibt es noch etwas, womit ich helfen kann? | weglassen |

### 7.3 Überstruktur in kurzen Nachrichten

**Verbot:** Durchstrukturierte Mails mit Zwischenüberschriften, fünf Bullets und drei Fettungen für eine simple Anfrage; Bullet-Listen, wo zwei Sätze reichen; Mails ohne konkreten Handlungsauftrag („Lassen Sie mich Ihre Gedanken wissen").

**Stattdessen:** E-Mails sind Prosa, Länge der Sache angemessen. Keine Überschriften unter etwa 150 Wörtern. Bullets nur für echte Aufzählungen (Termine, Optionen A/B/C). Genau eine klare Handlungsaufforderung pro Mail, mit Datum oder Deadline. In Mails und Slack reden wie am Telefon, nur aufgeräumt.

### 7.4 Konnektoren-Häufung

**Verbot/Tell:** „Zudem möchte ich anmerken ... Ferner ist zu beachten ... Darüber hinaus ..." in einer kurzen Mail ist ein starkes KI-Signal, ebenso durchgehend gestelzter Ton ohne einen einzigen lockeren Satz.

**Stattdessen:** „und", „außerdem", „noch was:". Maximal ein förmlicher Konnektor pro Mail.

### 7.5 Personalisierungs-Anker

**Verbot/Tell:** KI-Mails referenzieren keine gemeinsame Historie: kein Bezug aufs letzte Gespräch, keine Namen, keine Projektdetails, nur Oberflächen-Aussagen („Ihre Website hat großes Potenzial").

**Stattdessen:** Mindestens ein spezifischer Anker pro Mail: letzte Unterhaltung, konkrete Zahl aus dem Projekt, ein Detail, das nur wir wissen können („die 404-Seite auf /leistungen/ ist seit Dienstag gefixt").

### 7.6 Slack und Plaintext

**Verbot:** Markdown-Syntax, die als Symbole rendert (Sternchen, Rauten); Blockquote-Formatierung bei Texten, die kopiert werden sollen; Emoji-Bullets; förmliche Briefstruktur in Chat-Nachrichten.

**Stattdessen:** Kopierbare Texte als normale Absätze ausgeben. Chat-Ton: kurz, direkt, ohne Grußformel-Zeremonie.

## 8. Ton, Haltung und Inhalt

### 8.1 Überoptimismus und Superlative

**Verbot:** Durchgehend positive Grundtönung, jedes Produkt „revolutionär", jede Maßnahme „ein absolutes Muss", „Sie werden begeistert sein". Kritik fehlt oder kommt als mechanischer Pflicht-Herausforderungen-Absatz. Corporate-Pep-Talk ohne die frustrierenden Teile echter Arbeit.

**Stattdessen:** Superlative nur mit Beleg (Zahl, Fall, Quelle) und Geltungsgrenze (für wen gilt es nicht). Echte Nachteile konkret benennen. Ein ehrlicher Satz („Für Teams unter 5 Leuten lohnt sich das nicht") macht den Text glaubwürdiger als zehn Adjektive. Belegbare definitive Aussagen sind erlaubt und menschlich („war die erste", „ist die einzige").

### 8.2 Generische Beispiele und erfundene Belege

**Verbot:** Platzhalter-Personas („eine Studentin namens Anna", „ein mittelständisches Unternehmen", „Stellen Sie sich vor, Sie sind Bäcker"), erfundene Studien („Eine aktuelle Studie zeigt, dass 78 % ..."), vage Autorität („Viele Experten sind sich einig", „Studien belegen" ohne Quelle), erfundene Zahlen, Paragraphen, Normen und Kundenstimmen. Erfundene Patientenstimmen und absolute Heilungsaussagen sind im Medizinbereich zusätzlich ein HWG-Rechtsrisiko.

**Stattdessen:** Nur Zahlen mit nachprüfbarer Quelle (Name, Jahr) publizieren; eigene Tool-Daten (Analytics, Search Console, Ahrefs mit Datum) schlagen jede Fremdstudie. Mindestens ein echtes, prüfbares Detail pro Text: realer Fall, anonymisiert mit echten Zahlen („Ein Dortmunder Sanitärbetrieb, 12 Mitarbeiter: +38 % Anfragen in 4 Monaten"). Ohne echten Fall: keinen erfinden, Beispiel weglassen. Kundenstimmen nie generieren, nur echte, freigegebene Zitate.

### 8.3 Erfahrungs-Marker und Partikel

**Gebot:** Erste-Hand-Signale einbauen, die KI nicht fälschen kann: „Wir sehen bei Kunden oft ...", „Bei uns hat das erst funktioniert, als ...", dokumentierte Überraschungen und Fehlschläge („Wir hatten X erwartet, gemessen haben wir Y"). Deutsche Abtönungspartikel dosiert zulassen (eben, halt, nämlich, ohnehin, übrigens, schlicht, eigentlich); sie fehlen in KI-Deutsch fast völlig, weil sie im Englischen keine Entsprechung haben. Meinungen, gemischte Gefühle und Ich-Perspektive gehören in stimmtragende Textsorten; bei Doku, Fachreferenz und Recht ist neutral-schlicht die richtige menschliche Stimme.

### 8.4 Anrede- und Schreibweisen-Konsistenz

**Verbot:** Wechsel zwischen Du und Sie im selben Text oder Kanal (Hook im Du, CTA im Sie); gemischte Genderformen („Mitarbeiter:innen" neben „Mitarbeitende" neben generischem Maskulinum); uneinheitliche Schreibweisen (E-Mail/Email, Website/Webseite) im selben Dokument.

**Stattdessen:** Vor dem Schreiben festlegen: Anrede, Genderform, Schreibweisen-Glossar, pro Kunde und Kanal, passend zur restlichen Kommunikation (Website!). Konsistenz-Pass am Ende, inklusive CTA und Kommentar-Antworten.

### 8.5 Menschliche Signale bewahren, Über-Polieren vermeiden

**Gebot:** Diese Merkmale nie wegeditieren: schräge, schwer fälschbare Details; datierte, szenegebundene Referenzen; echte Einschübe und Selbstkorrekturen; feste Lieblingswendungen des Autors; ungelöste Spannung („überwiegend gut, aber ein Punkt stört mich"). Aggressives Wegschleifen jeder Unregelmäßigkeit drückt den Text zurück ins KI-Statistikprofil. Nicht jede Regel auf Maximum anwenden; gelegentliche natürliche Hedges („vielleicht", „eher") und schlichte Konstruktionen sind menschliche Marker. Ist ein Text schon sauber: unverändert lassen und das sagen (No-Op-Pfad).

## 9. KI-Selbstverräter (sofort fixen, vor allem anderen)

**Verbot:** Reste des Chat-Kontexts im publizierten Text. Jeder einzelne Fund ist ein Glaubwürdigkeits-Totalschaden:

- Chat-Präambeln und Chat-Korrespondenz: „Gerne! Hier ist ...", „Natürlich! Hier sind ...", „Hier ist Ihr überarbeiteter Text:", „Ich hoffe, das hilft", „Soll ich fortfahren?"
- Selbstreferenzen: „Als KI-Sprachmodell ...", „Stand meines letzten Wissensupdates", „Basierend auf den bereitgestellten Informationen"
- Wissenslücken-Prosa: „Über die Gründung ist wenig dokumentiert, vermutlich ..." (entweder recherchieren oder Satz streichen; Vermutung nie als Fakt verkleiden)
- Platzhalter: „[Firmenname]", „[Ihr Name]", „[CITY]", „XX", „TODO", Lorem ipsum
- Abgeschnittene Sätze am Token-Limit, doppelte Absätze, englische Restfragmente im deutschen Text
- Technische Fingerprints: `utm_source=chatgpt.com`, `oaicite`, `turn0search`, `contentReference`
- Sykophantie in Antworten: „Tolle Frage!", „Da haben Sie vollkommen recht!"

**Stattdessen:** Publikations-Checkliste vor jedem Versand: Suchlauf auf eckige Klammern, XX, TODO, utm_source, Markdown-Reste; letzten Satz auf Vollständigkeit prüfen; Text einmal im Zielformat rendern und ansehen.

## 10. Prüf-Workflow

### 10.1 Cluster-Regel

Kein einzelnes Muster beweist KI. Ein Gedankenstrich bedeutet nichts; Gedankenstriche plus Dreierregel plus „nahtlos" plus Fazit-Abschnitt sind ein Geständnis. Erst ab drei unabhängigen Pattern-Kategorien einen Text als KI-verdächtig behandeln oder umschreiben. Nicht flaggen: perfekte Grammatik allein, formelles Vokabular allein, einzelne Konnektoren, gemischte Register. Formelhaftes menschliches Schreiben triggert Leser genauso; diese Regeln gelten als Qualitätsstandard für jeden Text, egal wer den Entwurf geschrieben hat.

### 10.2 Prioritäten (bei Zeitdruck von oben nach unten)

1. **P0, Glaubwürdigkeits-Killer:** Chat-Artefakte, Platzhalter, erfundene Zahlen/Quellen/Zitate, Cutoff-Disclaimer, technische Fingerprints.
2. **P1, offensichtlicher KI-Geruch:** Bannwörter und Floskeln, Weltlage-Opener, Meta-Ankündigungen, Fettungs- und Bullet-Exzess, Gedankenstrich-Dichte, englische Typografie im deutschen Text.
3. **P2, Stilpolitur:** Dreierregel, uniforme Satz- und Absatzlängen, Kopula-Vermeidung, Konnektoren, generische Schlüsse.

### 10.3 Zwei-Pass-Pflicht

Erst Inhalt schreiben oder generieren, dann ein dedizierter De-Slop-Pass gegen diese Referenz. Nach dem ersten Rewrite die Selbstfrage stellen: „Was macht diesen Text noch offensichtlich KI-generiert?" Verbliebene Tells benennen und in einem zweiten Durchgang beheben. Erste Rewrites lassen regelmäßig Muster durch (recycelte Konnektoren, Rest-Inflation, zu aufgeräumter Rhythmus). Alle Regeln still anwenden, nie im Output erwähnen.

### 10.4 Schnelltests

- **Austauschbarkeits-Test:** Passt der Satz wortgleich auf jedes andere Unternehmen oder Thema? Dann konkretisieren oder streichen.
- **Laut-Lese-Test:** Würde ein Mensch das am Telefon oder in der Kaffeeküche so sagen? Stolpersätze umschreiben.
- **Streich-Test:** Ersten Absatz und jede Bedeutungs-Klausel probeweise löschen; fehlt nichts, bleibt es gelöscht.
- **Rückübersetzungs-Test:** Klingt der Satz auf Englisch natürlicher? Kalque, neu formulieren.
- **Konkretisierungs-Quote:** Pro 300 Wörter mindestens eine Zahl, ein Eigenname oder ein prüfbares Detail; pro Text mindestens eine Stelle mit Kante (was nicht funktioniert, für wen es sich nicht lohnt).
- **Dreier-Detektor:** Jede Aufzählung mit exakt drei Elementen prüfen und meist auf 2, 4 oder 5 ändern.

### 10.5 Suchlauf vor Abgabe

Mechanisch greppen, nicht nach Augenmaß: „—", englische Anführungszeichen, Emoji, eckige Klammern, „XX", „utm_source", Markdown-Sternchen, „Es ist wichtig", „In der heutigen", „Zusammenfassend", „nicht nur", „zudem", „darüber hinaus", Dezimalpunkte in Zahlen.

### 10.6 Rewrite statt Patchen

Bei 5 oder mehr Vokabular-Treffern über mehrere Kategorien plus 3 ausgelösten Pattern-Kategorien plus uniformem Rhythmus ist die Struktur selbst KI-generiert. Dann nicht flicken: Kernaussage in einem Satz formulieren und den Text von dort neu aufbauen.

### 10.7 Fakten sind unantastbar

Zahlen, Quellen, Fachbegriffe und technische Details bleiben beim De-Sloppen 1:1 erhalten; der Rewrite deckt alles ab, was das Original abdeckt. Keine Fake-Quirks oder absichtlichen Tippfehler injizieren; Menschlichkeit kommt aus Subtraktion und echter Spezifik. Zitierte Beispiele schlechten Schreibens (in Anführungszeichen oder Codeblöcken) werden nie mit-umgeschrieben.

### 10.8 Stimme kalibrieren

Liegt eine Schreibprobe des Autors oder der Marke vor (2 bis 3 echte Absätze, Mails, Posts): vor dem Rewrite Satzrhythmus, Wortniveau, Interpunktions-Gewohnheiten und wiederkehrende Wendungen analysieren und die KI-Muster durch die Muster der Probe ersetzen, nicht durch generisch Sauberes. Ohne Probe: natürlich, variiert, meinungsstark als Default. Wer „Zeug" schreibt, bekommt keine „Komponenten".

### 10.9 Bannlisten in Prompts

Wort-Verbote in Generierungs-Prompts müssen Synonyme explizit mitverbieten („keines dieser Wörter und keine sinngleichen Umschreibungen"), sonst ersetzt das Modell „überragend" nur durch „herausragend". Immer mit Positiv-Anweisung koppeln (konkrete Zahlen, kurze Sätze, aktive Verben) und den Output trotzdem per Suchlauf gegenprüfen. Wortlisten veralten mit den Modell-Generationen; die Struktur-Regeln (Rhythmus, Gliederung, Spezifik) veralten nicht und wiegen schwerer.
